## Ключевые поля для генерации нарядов

### FA_03 — Ведомость отправочных марок (уровень элемента)
```
Марка       → position (КС1-1, ВС-1, ...)
Кол-во      → quantity  
Наименование → element (Балка трубчатая, Колонна...)
Масса ед.   → unit_weight
Масса всего → total_weight
№ чертежа   → drawing_link (нужно смаппить на Drive)
RAL цвет    → для покраски
```

### FA_07 — Ведомость деталей (уровень детали внутри элемента)
```
Наименование → тип заготовки (Лист 100x4, Труба 80x4, Уголок 90x7...)
Кол-во       → quantity
Ширина/Высота/Длина → размеры для раскроя
Масса ед.    → unit_weight
Материал     → марка стали (C245, C235, 09Г2С)
Операция     → из наименования: Лист→ПЛАЗМА, Труба/Уголок/Швеллер→ПИЛА
```

### FA_04 — Технологическая карта (связь марки ↔ детали)
```
Марка → список деталей → материалы → нормы времени
Самый детальный файл, содержит всё
```

## Логика генерации нарядов

### Шаг 1 — Определяем специализацию по типу детали

| Тип детали | Специализация |
|-----------|---------------|
| Лист 4.0 мм, 8.0 мм (пластины) | ПЛАЗМА |
| Труба, Уголок, Швеллер, Балка (прокат) | ПИЛА |
| После закрытия пилы + плазмы | СБОРКА |
| После сборки | СВАРКА → СВЕРЛЕНИЕ → ПОКРАСКА |

### Шаг 2 — Структура нарядов (из FA_03 + FA_07)

```
Лист "ПЛАЗМА":
  Строка = одна деталь из FA_07 типа "Лист"
  position = марка родительского элемента
  element  = наименование детали
  quantity = кол-во деталей × кол-во марок
  unit_weight, total_weight

Лист "ПИЛА":
  Строка = одна деталь из FA_07 типа "Труба/Уголок/Швеллер"
  (аналогично)
```

### Шаг 3 — Привязка чертежей
- FA_08 даёт список листов чертежей
- FA_03 даёт номер чертежа для каждой марки
- При генерации нарядов → ссылка на PDF в Google Drive (папка Чертежи)
