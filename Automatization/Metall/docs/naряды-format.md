## Структура листа (на примере "Наряд по плазме")

### Шапка (строки 1–6)
```
Строка 1: "Наряд по плазме №1"                    Очередь: 4
Строка 2: Приказание директора ООО ... №...
Строка 3-5: Тарифная сетка плазмы:
            до 1 кг      → 1.25 руб/кг
            от 1 до 5    → 0.51 руб/кг
            от 5 до 10   → 0.222 руб/кг
            от 10 до 50  → 0.1 руб/кг
```

### Таблица (с строки 7)

| Колонка | Название | Пример |
|---------|----------|--------|
| A (0) | Отправочный элемент (марка) | КС4-1, ВС1, Б1... |
| B (1) | Позиция детали | Д1, Д2, П1... |
| C (2) | Кол-во | 7, 72, 6... |
| D (3) | Масса единицы (кг) | 87.69, 1.2... |
| E (4) | Масса всего (кг) | 613.83, 86.4... |
| F (5) | Стоимость / расценка | формула или пусто |
| G (6) | Итого к заработку (руб) | 61.383, 44.064... |
| H (7) | Выполнить / дата | 2026-05-21 |
| I (8) | Исполнитель / Подпись | Кириллов, Выполнено |

### Итоги (в конце таблицы)
```
Итого:     [кол-во]  [масса всего]  [сумма]
Выполнено: [кол-во]
1.         [подпись]
2.         [подпись]
```

## Логика генерации из КМД

```
FA_07 (Ведомость деталей) → для каждой детали:

  Лист 4мм, 8мм, пластины  →  лист "Наряд по плазме"
  Труба, Уголок, Швеллер   →  лист "Наряд по пиле"

FA_03 (Отправочные марки) → даёт:
  - Марку (отправочный элемент) для колонки A
  - Итоговую массу элемента
  - № чертежа

Тариф → из таблицы тарифов (уже в БД):
  payment_sum = unit_weight × кол-во × ставка(кг)
```

---

## Что генерируем автоматически

1. Читаем FA_03 + FA_07
2. Сортируем детали по типу → определяем лист (специализацию)
3. Заполняем строки таблицы
4. Рассчитываем payment_sum по тарифной сетке
5. Дату план и исполнителя — мастер заполняет вручную после загрузки

---

## Что остаётся ручным (мастер)

- Дата выполнения (date_plan) — мастер выставляет приоритет
- Исполнитель (executor) — мастер назначает рабочего
- Приоритет (обязательная / нет) — мастер решает
