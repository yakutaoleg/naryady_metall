# Бизнес-процесс — Цех металлоконструкций

*Источник: резюме интервью 27.05.2026*

---

## О клиенте

- Производство металлоконструкций под заказ: колонны, балки, фермы, прогоны
- Каждый проект уникальный (не серийное производство)
- Объём: 65–80 тонн в месяц
- ~30 рабочих в цеху

---

## Роли

| Роль | Что видит | Что делает |
|------|-----------|------------|
| Администрация | Все элементы, все блокировки, общий план загрузки | Загружает Excel (КМД) в систему. Принимает решения по блокировкам. |
| Мастер | Весь цех в реальном времени, статусы всех позиций | Расставляет приоритеты. Реагирует на блокировки. Видит сводку по смене. |
| Рабочий | Только свои позиции в порядке приоритета + чертёж | Отмечает статус. Сигнализирует о блокировках. |

---

## Специализации рабочих

| Специализация | Что делает |
|--------------|------------|
| ПИЛА | Профили, трубы, балки |
| ПЛАЗМА | Пластины из листа |
| СБОРКА | Сборка элемента (после закрытия пилы + плазмы) |
| СВАРКА | Сварка |
| СВЕРЛЕНИЕ | Сверление |
| ПОКРАСКА | Покраска |

---

## Бизнес-процесс (целевое состояние)

1. КМД формирует список элементов → передаёт Администрации
2. Администрация загружает список в систему (Excel → Google Sheets)
3. Мастер расставляет приоритеты по позициям
4. Рабочие видят задачи строго по приоритету
5. По каждой позиции рабочий проставляет статус
6. Система проверяет: все детали элемента готовы?
   - НЕТ → элемент остаётся в работе
   - ДА → элемент переходит на сборку
7. Сборка → Сварка → Сверление → Покраска → Отгрузка

**Ключевое правило:** Отправочный элемент переходит на сборку только когда ВСЕ его детали готовы на пиле И на плазме. Пока хоть одна позиция не закрыта — элемент стоит.

---

## Статусы позиций

| Статус | Кто видит | Что происходит |
|--------|-----------|----------------|
| ПЛАН | Рабочий | Задача в очереди |
| ВЫПОЛНЕНО | Мастер, система | Позиция закрыта. Система пересчитывает готовность элемента. Стоимость добавляется к выработке рабочего. |
| НЕ ВЫПОЛНЕНО + причина | Мастер | Возвращается в список рабочего. Причина обязательна. |
| ЗАБЛОКИРОВАНО + причина | Мастер + Администрация | Немедленное уведомление. Ждёт решения сверху — не возвращается автоматически. |

---

## Боли клиента

| Приоритет | Боль | Описание |
|-----------|------|----------|
| ГЛАВНАЯ | Слепые зоны в производстве | Плазморезчик не нарезал деталь — узнают только когда сборщик уже стоит. |
| ГЛАВНАЯ | Рабочие скрывают проблемы | Не отметили, промолчали. Системная проблема. |
| ВАЖНАЯ | Работа не по приоритету | Выбирают выгодные позиции, пропускают нужные. |
| ВАЖНАЯ | Бумажные наряды | Теряются, нет преемственности между сменами. |
| ВАЖНАЯ | Нет зависимостей между этапами | Сборщик начинает на элементе у которого не все детали готовы. |
| СИСТЕМНАЯ | Невозможно планировать загрузку | Руководство не знает когда цех освободится. |

---

## Защита от работы не по порядку

- Рабочий видит позиции строго в порядке приоритета
- Приоритет выставляет мастер вручную
- Необязательные задачи заблокированы пока есть незакрытые обязательные
- При нажатии на заблокированную задачу — сообщение "Сначала выполните обязательные"
