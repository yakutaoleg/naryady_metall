# Telegram Bot — UX и логика работы

## Концепция

Бот для рабочих и мастеров цеха металла. Всё управление — только через кнопки, без команд.
Рабочий обычно имеет одну специализацию и видит только свои задачи.
Мастер/Администратор — видит весь цех.

---

## Экраны рабочего

### 1. Главное меню (`/start`)

```
👷 ГАНЖА Иван
🔧 Специализация: ПЛАЗМА

Выберите действие:

[ 📋 Задачи в работу ]  [ ✅ Выполнено сегодня ]
```

### 2. Список задач

```
📋 Задачи | ПЛАЗМА

❗ Обязательные:
  • ПС1 — СТ1 × 8 шт
  • ПС5 — СТ1-А × 2 шт

Остальные 🔒 (после обязательных):
  • ПС2 — СТ1 × 4 шт
  • ПС7 — Р1 × 1 шт

👇 Нажмите на задачу:
[ ❗ ПС1 — СТ1 × 8     ]
[ ❗ ПС5 — СТ1-А × 2   ]
[ 🔒 ПС2 — СТ1 × 4     ]
[ 🔒 ПС7 — Р1 × 1      ]
[ ← Главное меню        ]
```

**Логика блокировки:**
- Пока есть невыполненные обязательные — необязательные заблокированы (🔒)
- При нажатии на 🔒 задачу: pop-up "Сначала выполните обязательные: ПС1, ПС5"
- Как только все обязательные выполнены — 🔒 снимается автоматически

### 3. Карточка задачи

```
❗ [ПЛАЗМА] ПС1
Элемент: СТ1
Проект: Орша теплосети
Кол-во: 8 шт
Масса ед.: 1.47 кг  |  Масса всего: 11.76 кг
Дата план: 20.06.2026
Обязательная: ДА

💰 Тариф: 1.25 руб/кг
💵 К оплате: 5.99 руб

[ ✅ ВЫПОЛНЕНО ]  [ 🚫 БЛОК ]
[ 📄 Чертёж               ]   ← только если прикреплён
[ ← К задачам             ]
```

### 4. После ВЫПОЛНЕНО

Статус задачи → ВЫПОЛНЕНО, дата факт → сегодня.
Бот сразу возвращает к обновлённому списку задач.

**Защита от двойного нажатия:** повторный клик ВЫПОЛНЕНО показывает alert "Задача уже отмечена как выполненная."

### 5. БЛОК

Бот просит написать причину текстом → сохраняет в поле `comment`.
После → кнопка "← К задачам".

**Доступ:** рабочий может поставить БЛОК только на свою задачу (проверка `executor`).

### 6. Чертёж

Рабочий нажимает "📄 Чертёж" → бот скачивает PDF из Google Drive и отправляет в чат.

**Доступ:** чертёж доступен только исполнителю задачи (проверка `executor`).

### 7. Выполнено сегодня

```
✅ Выполнено сегодня (2):

✅ ПС1 — СТ1 × 8 шт  |  5.99 руб
✅ ПС5 — СТ1-А × 2 шт  |  3.20 руб

💵 Итого за день: 9.19 руб

[ ← Главное меню ]
```

## Экраны мастера / администратора

### 1. Главное меню мастера

```
👷 ИВАНОВ Пётр  |  Мастер
Проект: Орша теплосети

[ 🚫 Блоки ]         [ 📊 Выработка ]
[ 📁 Проекты ]       [ 🔄 Синхронизировать ]

Последняя синхронизация: 14:32
```

### 2. Экран "🚫 Блоки"

Все активные блокировки по всем проектам с кнопкой "Снять блок".

**Защита от двойного снятия:** повторный клик показывает alert "Блок уже снят другим мастером."

### 3. Экран "📊 Выработка"

```
📊 Выработка — Неделя

Иванов А.      — 1 240.50 руб
Петров В.      —   980.00 руб
Сидоров К.     —   760.20 руб

[ Сегодня ]  [ Неделя ]  [ Месяц ]
[ ← Назад ]
```

Фильтр по периоду: Сегодня / Неделя / Месяц.

### 4. Управление проектами

Добавить проект по ссылке Google Drive, архивировать, восстановить.

### 5. Исправить выполнение

**Доступ:** только мастер или администратор

Мастер может корректировать задачи, выполненные сегодня или вчера. Используется когда рабочий неправильно выполнил или переоценил количество.

#### Флоу:

```
1️⃣ Главное меню → [🔧 Исправить выполнение]
   Показывается список рабочих, у которых есть ВЫПОЛНЕНО или ЧАСТИЧНО за сегодня/вчера

2️⃣ Выбор рабочего → список его задач
   Группировка: Проект → Специализация → статусы ВЫПОЛНЕНО / ЧАСТИЧНО

3️⃣ Выбор задачи → меню действий (зависит от статуса)

4️⃣ Выбор действия → сохранение изменений
```

#### Сценарии исправления:

| Исходный статус | Действие | Результат | Описание |
|---|---|---|---|
| **ВЫПОЛНЕНО** | "Отменить полностью" | Статус → ПЛАН<br/>qty_done = NULL<br/>date_fact = NULL | Задача вернулась в очередь рабочего без изменения запланированного количества |
| **ВЫПОЛНЕНО** | "Исправить количество" | Статус остаётся ВЫПОЛНЕНО<br/>qty_done уменьшается<br/>Остаток → новая строка ПЛАН без исполнителя | Создаётся новая задача для оставшегося количества |
| **ЧАСТИЧНО** | "Отменить полностью" | Статус → ПЛАН<br/>qty = done + остаток<br/>Строка остатка удаляется | Объединение двух строк в одну исходную задачу |
| **ЧАСТИЧНО** | "Исправить количество" | qty_done уменьшается<br/>Остаток обновляется | Пересчёт обеих частей |

#### Граничные случаи:

- **qty=1:** показывается только кнопка "Отменить полностью" (исправлять нельзя)
- **Остаток уже выполнен:** если ЧАСТИЧНО имеет остаток, который уже выполнили сегодня, исправление блокировано с пояснением "Сначала исправьте сегодняшнее выполнение остатка"
- **Сохранение date_fact:** при исправлении оригинальная дата рабочего не меняется
- **Новый остаток:** после исправления без исполнителя, статус ОБЯЗАТЕЛЬНАЯ=НЕТ (необязательная)

#### Пример экрана:

```
Исправить выполнение

🔧 Рабочий: СИЗОВА Елена | ПЛАЗМА

[ 📊 НеПроект — Орша теплосети ]

✅ ПС1 — СТ1 × 8 шт  (выполнено: 8)
ЧАСТИЧНО: ПС5 — СТ1-А  (выполнено: 2 из 4)

👇 Выберите задачу для исправления:
[ ✅ ПС1 — СТ1 × 8 ]
[ ЧАСТИЧНО ПС5 — СТ1-А ]
[ ← Назад ]
```

При выборе задачи:

```
ПС1 — СТ1 × 8 шт
Статус: ✅ ВЫПОЛНЕНО
Выполнено: 8 шт | Дата: 20.07.2026

[ ↩️ Отменить полностью ]
[ ✏️ Исправить количество ]
[ ← Назад ]
```

Если выбрать "Исправить количество" → бот запрашивает новое количество (число) → сохраняет и создаёт остаток.

## Автоматическая разблокировка зависимостей

### Концепция

В производстве существует цепочка технологических зависимостей: задачи одной специализации блокируют задачи следующей.

```
ПЛАЗМА / ПИЛА  →  СБОРКА  →  СВАРКА  →  ПОКРАСКА
(Резка)          (Сборка)    (Сварка)    (Отделка)
```

Когда предыдущий этап выполнен полностью или частично — следующий этап автоматически разблокируется.

### Хранение зависимостей

Цепочка зависимостей хранится в таблице БД `element_dependencies`:
- `element_id` — элемент (например, Ф-1, Ф-2)
- `from_specialization` — откуда блокировка (ПЛАЗМА → СБОРКА)
- `to_specialization` — куда разблокировка (СБОРКА → СВАРКА)

Данные синхронизируются из Google Sheets при каждом обновлении проекта.

### Сценарий разблокировки

#### Пример: рабочий выполнил СБОРКА частично

```
1️⃣ Рабочий отмечает СБОРКА Ф-1 как ЧАСТИЧНО:
   Выполнено: 4 из 5 шт

2️⃣ Бот автоматически ищет в таблице:
   - Кто ждёт Ф-1? → СВАРКА Ф-1 × 5 в статусе БЛОК
   
3️⃣ Разбивает заблокированную строку:
   Было: СВАРКА Ф-1 × 5 (БЛОК)
   
   Стало:
   - СВАРКА Ф-1 × 4 (ПЛАН) — готово к выполнению
   - СВАРКА Ф-1 × 1 (БЛОК) — ещё ждёт остатка от СБОРКИ
   
   Комментарий в новой строке БЛОК: "Ожидает готовности остатка"

4️⃣ Исполнитель копируется в обе строки (тот же рабочий)

5️⃣ Обновляются одновременно:
   - БД (таблицы `tasks` и `task_rows`)
   - Google Sheets (проектный лист)
```

#### Пример: рабочий выполнил всё полностью

```
Рабочий отмечает СБОРКА Ф-1 как ВЫПОЛНЕНО: 5 из 5 шт

↓

Система находит СВАРКА Ф-1 × 5 (БЛОК)

↓

Полная разблокировка:
Статус меняется БЛОК → ПЛАН (без разбиения)
```

### Уведомления при разблокировке

#### Рабочему:

```
✅ Блокировка снята
───────────────────
Позиция: Ф-1
Специализация: СВАРКА

Ты можешь начинать работу!
```

**Когда:** Если рабочему назначена разблокированная задача (executor совпадает)

#### Мастерам:

```
🔔 Автоматическая разблокировка
───────────────────────────────
Проект: Орша теплосети
Специализация: СВАРКА

СВАРКА Ф-1:
  ✅ 4 шт → ПЛАН (готово)
  🔒 1 шт → ещё заблокировано
```

**Когда:** Для всех активных мастеров проекта

### Граничные случаи

| Сценарий | Поведение |
|----------|-----------|
| **qty_done > qty_blocked** | Разбиение: qty_blocked идёт в ПЛАН, хвост остаётся в БЛОК |
| **qty_done = qty_blocked** | Полная разблокировка (просто БЛОК → ПЛАН) |
| **qty_done < qty_blocked** | Разбиение: qty_done в ПЛАН, остаток в новый БЛОК |
| **Executor не найден** | Исполнитель не копируется в новый БЛОК (необязательная задача) |
| **Нет зависимостей** | Разблокировка не происходит, функция завершается тихо |

## Идентификация

- Поиск по `telegram_username` в таблице `employees`
- Специализация берётся из `employees.specialization`
- Роль (worker / master / admin) из `employees.role`
- Если пользователь не найден — бот просит обратиться к руководителю

## Тарифы

Хранятся в таблице `tariffs` (синхронизируются из вкладки ТАРИФЫ в Google Sheets).

| Вид работы | Диапазон массы ед. (кг) | Ставка      |
|------------|--------------------------|-------------|
| ПЛАЗМА     | 0 — 1                    | 1.25 руб/кг |
| ПЛАЗМА     | 1 — 5                    | 0.51 руб/кг |
| ПЛАЗМА     | 5 — 10                   | 0.222 руб/кг|
| ПЛАЗМА     | 10+                      | 0.10 руб/кг |
| ПИЛА       | любая                    | 0.15 руб/кг |
| СБОРКА     | любая                    | 0.25 руб/кг |
| СВАРКА     | любая                    | 0.234 руб/кг|
| ПОКРАСКА   | любая                    | 0.12 руб/кг |

Сумма к оплате (`payment_sum`) берётся из Google Sheets как есть.
Тариф в карточке задачи — справочная информация.

## Защита статусов

Sync из Google Sheets **не перезаписывает** статусы ВЫПОЛНЕНО и БЛОК.
Только сам рабочий через бот может их установить.
