# Metall Bot — Управление производством металлоконструкций

Telegram-бот для организации работы цеха металлоконструкций. Позволяет рабочим отслеживать задачи по мере их выполнения, а мастерам управлять процессом в реальном времени.

## Возможности

### Для рабочих
- 📋 Просмотр задач строго по приоритету (обязательные → остальные)
- ✅ Отметить выполнение с сохранением даты и количества
- 🚫 Сигнализировать о блокировках с описанием причины
- 📄 Скачать чертёж детали прямо в чате
- 💵 Смотреть свою выработку за день/неделю/месяц
- 🔐 Доступ только к своим задачам и чертежам

### Для мастеров
- 👀 Полный обзор цеха в реальном времени
- 🚫 Управление блокировками (просмотр, снятие, анализ причин)
- 📊 Выработка по рабочим за день/неделю/месяц
- 🔧 **Исправление выполнения** — корректировка ошибок рабочих за сегодня/вчера
- 🔄 Синхронизация с Google Sheets (двусторонняя)
- 📁 Управление проектами и их чертежами

### Система статусов
| Статус | Иконка | Описание |
|--------|--------|---------|
| ПЛАН | 📋 | Задача в очереди |
| ВЫПОЛНЕНО | ✅ | Готово, добавлено в выработку |
| ЧАСТИЧНО | 🌗 | Выполнено часть, остаток в ПЛАН |
| БЛОК | 🚫 | Задача заблокирована, ждёт решения мастера |

## Роли в системе

| Роль | Видит | Может делать |
|------|-------|-------------|
| **Рабочий** | Только свои задачи | Отметить выполнено, сигнализировать блокировку |
| **Мастер** | Весь цех | Управлять приоритетами, снимать блоки, исправлять ошибки |
| **Администратор** | Всё | Всё как мастер + управление проектами и сотрудниками |

## Ключевые особенности

### Синхронизация с Google Sheets
- Google Sheets — источник истины для статусов
- При изменении статуса в Sheets → БД обновляется автоматически
- Защита: бот не перезаписывает статусы которые он сам установил
- Кэширование для производительности

### Защита от ошибок
- Рабочий видит задачи строго по приоритету (нельзя пропустить обязательные)
- Мастер может исправить ошибку выполнения за сегодня/вчера
- Двойные клики на кнопки защищены (alert при повторном нажатии)
- Все операции логируются

### Зависимости между этапами
- Элемент переходит на следующий этап (сборка) только когда ВСЕ его детали готовы
- Автоматическое определение готовности на основе специализаций
- Уведомления для сборщиков когда элемент готов
