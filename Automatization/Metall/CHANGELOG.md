## [2026-07-21] — Автоматическая разблокировка зависимых задач

### Added
- Функция `notify_deps_unblocked(bot, project_name, completed_sheet, completed_position, qty_done)` в `src/bot.py`
  - Вызывается когда задача переходит в ВЫПОЛНЕНО или ЧАСТИЧНО
  - Автоматически разбивает блокированные задачи следующего уровня зависимостей
  - Обновляет БД и Google Sheets одновременно
- Функция `_notify_worker_unblocked(bot, executor, position, element)` — отправляет рабочему уведомление "✅ Блокировка снята" когда его задача разблокирована
- Функция `_notify_masters_dep_unblocked(bot, project_name, waiting_sheet, position, qty_unblocked, qty_still_blocked)` — уведомляет всех мастеров об автоматической разблокировке с деталями количества

### Features
- **Цепочка зависимостей:** ПЛАЗМА/ПИЛА → СБОРКА → СВАРКА (хранится в таблице `element_dependencies`)
- **Автоматическое разбиение:** когда рабочий отмечает СБОРКА как ЧАСТИЧНО (например, 4 из 5)
  - Система находит в таблице кто ждёт эту позицию (СВАРКА × 5 в БЛОК)
  - Разбивает: СВАРКА Ф-1 × 5 → СВАРКА Ф-1 × 4 (ПЛАН) + СВАРКА Ф-1 × 1 (БЛОК)
  - Добавляет комментарий в разблокированную строку: "Ожидает готовности остатка"
- **Наследование исполнителя:** при разбивке executor копируется в обе строки (ПЛАН и новый БЛОК)
- **Полная разблокировка:** если qty_done >= qty_blocked, строка не разбивается — просто статус БЛОК → ПЛАН
- **Уведомления:**
  - Рабочему: "✅ Блокировка снята — Позиция: Ф-1"
  - Всем мастерам: "🔔 Автоматическая разблокировка — СВАРКА Ф-1: 4 шт → ПЛАН, 1 шт → ещё заблокировано"

### Integration Points
- Вызывается из `done:` handler когда задача полностью выполнена
- Вызывается из `receive_partial_qty` когда задача отмечена как ЧАСТИЧНО

## [2026-07-20] — Функция "Исправить выполнение"

### Added
- Новый экран мастера **"🔧 Исправить выполнение"** в главном меню
- Функция `delete_row(file_id, sheet_name, row_num)` в `sheets.py` — удаление строк из Google Sheets
- 5 новых callback-обработчиков для исправления задач
- Поддержка ConversationHandler с состоянием `WAITING_CORRECTION_QTY`

### Changed
- Главное меню мастера: добавлена кнопка "🔧 Исправить выполнение" после "🚫 Блоки"

### Features
- **Отмена выполнения:** мастер может вернуть выполненную задачу в статус ПЛАН
- **Исправление количества:** если рабочий неправильно оценил объём, мастер может уменьшить qty_done и создать остаток
- **Работа с ЧАСТИЧНО статусом:** исправление задач, которые выполнены частично
- **Граничные случаи:**
  - qty=1 → только отмена (исправить нельзя)
  - Если остаток уже выполнен → исправление блокировано
  - Сохранение оригинальной даты выполнения
  - Новые остатки создаются без исполнителя (необязательные)

### Access Control
- Только мастер или администратор имеют доступ к функции
- Только задачи за сегодня или вчера доступны для исправления

## [2026-07-16] — Исправление кэша Google Sheets

### Fixed
- Критический баг в `sheets.py`: кэш `_ss_cache`, `_ws_cache`, `_hdr_cache` не учитывал флаг `rw`
  - Причина: при открытии файла сначала RO, потом RW, возвращался кэшированный RO объект → 403 ошибка на запись
  - Решение: ключи кэшей теперь включают флаг `rw` в кортеже
  - Функция `_invalidate_ws` теперь очищает обе версии (rw=False и rw=True)
