# Скилл: Создание нового наряда

## Когда использовать

Пользователь даёт:
- Ссылку на **оригинальный файл** (исходные данные, несколько подтаблиц на листе)
- Ссылку на **пустой шаблон наряда** (уже отформатированный файл)
- Название подтаблицы / диапазон осей (например "Ось 4-10")

## Алгоритм

### Шаг 1 — Изучить оригинальный файл

Для каждого листа (ПЛАЗМА, ПИЛА, СВЕРЛЕНИЕ, СБОРКА, СВАРКА):
1. Прочитать все строки через `ws.get_all_values()`
2. Найти все строки где встречается `'ПОЗ. СОГЛАСНО ЧЕРТЕЖА'` — каждая такая строка = начало подтаблицы
3. Запомнить индекс колонки где найден заголовок → это `pos_col` этой подтаблицы
4. Читать заголовки той же строки правее → строить `col_map` по названию

### Шаг 2 — Определить колонки динамически

Для каждой подтаблицы строить словарь:
```python
col_map = {h.strip(): idx for idx, h in enumerate(header_row) if h.strip()}
```

Искать по названиям (регистронезависимо, частичное совпадение):
- `pos_col`       → 'ПОЗ. СОГЛАСНО ЧЕРТЕЖА'
- `name_col`      → содержит 'ПОЗ' НО является описанием элемента (следующая колонка после pos_col, если она не числовая)
- `qty_col`       → 'КОЛ-ВО' или 'ВСЕГО' или 'ЭЛЕМЕНТОВ'
- `mass_col`      → 'МАССА' + 'ЕД' или 'ЭЛЕМЕНТ' (масса единицы)
- `totalmass_col` → 'МАССА ВСЕХ' или 'МАССА' без 'ЕД'
- `pay_col`       → 'СУММА' или 'ОПЛАТ'

### Шаг 3 — Определить тип листа

- **Есть отдельная колонка с названием элемента** (следующая после pos_col содержит текст типа "пластина 120х60х4") → тип A (ПЛАЗМА, ПИЛА)
  - B = название элемента, C = позиция (ЭЛЕМЕНТ)
- **Нет колонки с названием** (сразу после позиции идут числа) → тип B (СБОРКА, СВАРКА)
  - B = позиция, нет колонки ЭЛЕМЕНТ

### Шаг 4 — Извлечь данные

```
для каждой строки начиная с data_start:
  - если pos_col непустой → обновить current_pos (carry forward)
  - если name_col пустой (тип A) → пропустить
  - если qty == 0 и total_mass == 0 → пропустить (позиция не входит в эти оси)
  - если в name/pos есть 'ПОЗ.' / 'ЧЕРТЕЖ' / 'КОМПЛЕКТ' → это заголовок, пропустить
  - иначе → добавить строку
```

Числа конвертировать через `float(val.replace(',', '.'))` → Python float.

### Шаг 5 — Записать в шаблон

```python
ws_dest.update(
    values=rows,
    range_name=f'A3:N{2 + len(rows)}',
    value_input_option='USER_ENTERED'   # ОБЯЗАТЕЛЬНО — иначе апострофы
)
```

### Шаг 6 — Добавить строку ИТОГО

После данных: пустая строка, затем ИТОГО с формулами `=SUM(...)`.

Для типа A (есть ЭЛЕМЕНТ): суммировать D (КОЛ-ВО), F (МАССА ВСЕХ), G (СУММА К ОПЛАТЕ)
Для типа B (нет ЭЛЕМЕНТ): суммировать C (КОЛ-ВО), E (МАССА ВСЕХ), F (СУММА К ОПЛАТЕ)

### Шаг 7 — Скопировать форматирование

```python
# copyPaste: строка 3 → все строки с данными
spreadsheet.batch_update({"requests": [{
    "copyPaste": {
        "source": {"sheetId": ws.id, "startRowIndex": 2, "endRowIndex": 3, ...},
        "destination": {"sheetId": ws.id, "startRowIndex": 2, "endRowIndex": last_row, ...},
        "pasteType": "PASTE_FORMAT"
    }
}]})
```

### Шаг 8 — Отформатировать строку ИТОГО

```python
# repeatCell: серый фон + жирный текст на строку ИТОГО
{"repeatCell": {
    "cell": {"userEnteredFormat": {
        "backgroundColor": {"red": 0.851, "green": 0.851, "blue": 0.851},
        "textFormat": {"bold": True}
    }},
    "fields": "userEnteredFormat(backgroundColor,textFormat)"
}}
```

## Частые ошибки

| Ошибка | Причина | Решение |
|--------|---------|---------|
| Числа с апострофом (`'5`) | `value_input_option` не указан (RAW по умолчанию) | Всегда `USER_ENTERED` |
| ИТОГО перекрыто данными | `batch_clear` захватил строку ИТОГО | Искать ИТОГО по тексту, не по номеру строки |
| ИТОГО жёлтая (как обычные строки) | `copyPaste` скопировал формат на ИТОГО тоже | Отдельным `repeatCell` перекрасить после |
| Пустая строка перед ИТОГО жёлтая | То же | `repeatCell` с белым фоном на строку N-1 |
| СБОРКА/СВАРКА Ось 11-15 = 0 строк | Неверный `pos_col` (смещение) | Читать header динамически |

## Инструмент запуска

```bash
python Automatization/Metall/create_naryad.py \
  --source <SPREADSHEET_ID_оригинала> \
  --target <SPREADSHEET_ID_шаблона> \
  --label "Ось 4-10"
```

`--label` нужен только для логов. Скрипт сам находит нужную подтаблицу по порядку:
- Первая встреченная `ПОЗ. СОГЛАСНО ЧЕРТЕЖА` → главная таблица (пропускаем)
- Вторая → первая подтаблица (Ось 4-10)
- Третья → вторая подтаблица (Ось 11-15)

Либо указать `--subtable 2` (номер подтаблицы, 1-based).
